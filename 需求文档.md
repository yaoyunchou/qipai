## 1. 项目概述

### 1.1 产品定位

云端网页版棋牌室**记账开单**系统：前后端分离，部署于公网云服务器，支持手机/电脑浏览器登录使用。

### 1.2 核心原则

| 原则 | 说明 |
|------|------|
| 纯记账 | 无计时、无自动计费、无真实收款 |
| 开单归档 | 前台开单 → 结单清台 → 订单永久留存 |
| 报表统计 | 日/周/月报表，支撑对账与远程查账 |
| 权限分层 | 四角色固定权限，超管可细粒度开关部分能力 |

### 1.3 使用场景

- 门店日常开单记账（收银员）
- 经理对账、全店单据与报表查看
- 财务/股东远程查看营收汇总
- 店主系统配置（台位、基准价、账号与权限）

### 1.4 技术栈（参考，非本期功能范围）

| 层级 | 选型 |
|------|------|
| 前端 | React |
| 后端 | Python FastAPI |
| 数据库 | MySQL |
| 部署 | 公网云服务器，独立域名访问 |

## 2. 系统角色与权限

系统**仅保留 4 个角色**，权限严格固定；超级管理员可在后台配置部分开关型权限。

### 2.1 收银员（一线开单）

| 能力 | 支持 |
|------|------|
| 前台开台、清台（结单归档） | ✅ |
| 固定原价开单 / 自定义价格开单 | ✅（是否允许改价由超管开关） |
| 查看自己的开单记录 | ✅ |
| 查看日报、周报、月报 | ✅（日期范围由超管授权，如仅近 2/3 个月或不限制） |
| 后台配置（加台、改台、改价、建账号） | ❌ |
| 查看他人单据、系统设置 | ❌ |

### 2.2 门店经理

| 能力 | 支持 |
|------|------|
| 收银员全部能力（开台、清台、原价/自定义价开单） | ✅ |
| 查看全店所有订单明细 | ✅ |
| 完整日/周/月营业报表 | ✅ |
| 报表导出 Excel | ✅ |
| 台位增删改、基准价格、新建账号 | ❌ |

### 2.3 股东（财务查看）

| 能力 | 支持 |
|------|------|
| 开台、清台、记账 | ❌ |
| 订单明细 | ❌（完全不可见） |
| 全店财务营收报表（日/周/月汇总） | ✅ |
| 报表导出 Excel | ✅ |
| 任何后台配置 | ❌ |

### 2.4 超级管理员（店主/最高权限）

| 能力 | 支持 |
|------|------|
| 系统全部功能 | ✅ |
| 前台任意开单、清台、查所有单据与报表 | ✅（可配置报表日期筛选上限：如一周内、一月内或不限制） |
| 台位管理：新增、修改、删除、启用/禁用 | ✅ |
| 基准价格修改（**仅对新单生效**，历史订单价格锁定） | ✅ |
| 用户账号：新增、删除、禁用各角色 | ✅ |
| 权限开关（如收银员改价、导出等） | ✅ |

### 2.5 权限矩阵（汇总）

| 功能模块 | 收银员 | 门店经理 | 股东 | 超级管理员 |
|----------|:------:|:--------:|:----:|:----------:|
| 前台开台/清台 | ✅ | ✅ | ❌ | ✅ |
| 自定义实收价开单 | 可配置 | ✅ | ❌ | ✅ |
| 查看本人订单 | ✅ | ✅ | ❌ | ✅ |
| 查看全店订单明细 | ❌ | ✅ | ❌ | ✅ |
| 本人相关报表 | ✅ | ✅ | ❌ | ✅ |
| 全店报表 | 受限日期 | ✅ | 仅汇总 | ✅ |
| 报表导出 Excel | 可配置 | ✅ | ✅ | ✅ |
| 台位/基准价/账号管理 | ❌ | ❌ | ❌ | ✅ |
| 权限开关配置 | ❌ | ❌ | ❌ | ✅ |

| 角色 | 登录后默认落地页 |
|------|------------------|
| 收银员 | **前台开单页面** |

| 股东 | **仅报表页面**（无其他操作菜单） |
| 超级管理员 | **后台首页**（含前台开单入口） |

## 4. 前台开单（核心业务）

### 4.1 业务边界

- **无**计时、**无**自动计费、**无**收款（微信/支付宝等）
- 每笔订单为人工开单记账，清台后归档

### 4.2 桌台状态

| 状态 | 展示 | 操作 |
|------|------|------|
| 空闲 | 绿色 | 可点击【开单】 |
| 占用 | 红色 | 可点击【结单清台】归档 |

### 4.3 开单规则

1. 开单时默认带出后台配置的**标准基准价**。
2. 支持手动修改为**自定义实收价**（收银员是否允许改价由超管配置）。
3. 每笔订单永久保存字段至少包括：
   - 原价（基准价）
   - 实收价
   - 开单人员
   - 开单时间
   - 备注（如有）
4. **清台**后：订单归档，桌台恢复为空闲。

### 4.4 数据可见性（订单明细）

| 角色 | 可见范围 |
|------|----------|
| 收银员 | 仅本人开的单据 |
| 门店经理、超管 | 全店所有人员单据 |
| 股东 | **不可见**任何订单明细 |

---

## 5. 后台管理（仅超级管理员）

### 5.1 台位管理

- 新增桌台、修改桌台名称、启用/停用桌台
- **删除限制**：存在未结订单的桌台**禁止删除**

### 5.2 基准价格管理

- 设置全场默认单价（基准价）
- 修改基准价**不影响**历史订单，**仅对新开订单**生效

### 5.3 账号与权限管理

- 新建四类角色账号，禁用/删除账号
- 可配置开关示例：
  - 收银员是否允许自定义改价
  - 报表导出权限
  - 收银员报表可查询的日期范围（如 2 个月 / 3 个月 / 不限制）
  - 超管报表日期筛选上限

---

## 6. 报表模块

### 6.1 统计维度

统一提供**日报、周报、月报**，统计指标至少包括：

| 指标 | 说明 |
|------|------|
| 总开单量 | 订单笔数 |
| 原价总营收 | 按基准价汇总 |
| 自定义实收总营收 | 按实收价汇总 |

### 6.2 各角色报表能力

| 角色 | 数据范围 | 订单明细 | 导出 Excel |
|------|----------|----------|------------|
| 收银员 | 与本人相关的报表数据 | 仅本人单据（开单模块） | 可配置 |
| 门店经理 | 全店完整报表 | 全店明细（开单模块） | ✅ |
| 股东 | 全店营收**汇总** | ❌ 无明细 | ✅ |
| 超级管理员 | 全店完整报表 | 全店明细 | ✅ |

### 6.3 报表日期范围

- 收银员：由超管授权可查询的历史跨度（如 2 个月、3 个月或不限制）
- 超级管理员：可配置自身或全局的报表筛选时间上限

---

## 7. 明确不做（范围外）

以下能力**永久不纳入**本系统，评审与开发均不得扩展：

- 计时 / 自动计费
- 微信、支付宝等真实收款
- 会员、押金、套餐、包场
- 库存管理

## 8. 部署与使用

| 项 | 要求 |
|----|------|
| 部署方式 | 前后端部署于云服务器，公网独立网址 |
| 终端 | 电脑、手机浏览器随时登录 |
| 远程场景 | 异地股东、远程经理可查看报表（股东仅汇总） |

## 9. 非功能需求（建议）

| 类别 | 建议 |
|------|------|
| 安全 | 账号密码登录；HTTPS；按角色鉴权 |
| 数据 | 订单归档后不可篡改价格；关键操作留痕（开单人、时间） |
| 可用性 | 前台桌台状态一眼可辨（绿/红）；移动端可完成开单与清台 |
| 导出 | Excel 导出字段与报表指标一致，经理/股东/超管按权限开放 |

## 10. 待确认项（开发前建议对齐）

| 编号 | 问题 | 建议默认 |
|------|------|----------|
| Q1 | 一单是否仅对应一桌、是否支持加钟/续单合并 | 一桌一单，清台即结案 |
| Q2 | 自定义实收价是否允许低于基准价 | 允许，由超管开关收银员改价权 |
| Q3 | 多门店 / 多租户 | 源文档未提及，默认**单店单实例** |
| Q4 | 订单备注是否必填 | 默认选填 |
| Q5 | 股东「无订单明细」是否包含导出中的明细 sheet | 仅汇总 sheet，无明细列 |

## 附录 A：术语

| 术语 | 含义 |
|------|------|
| 开台 / 开单 | 为空闲桌台创建一笔记账订单 |
| 清台 / 结单 | 结束当前桌台占用，订单归档，桌台变空闲 |
| 基准价 | 后台配置的标准单价，开单默认带出 |
| 实收价 | 开单时实际记账金额，可等于或不同于基准价 |
| 归档 | 清台后订单进入历史，不可再改价 |
